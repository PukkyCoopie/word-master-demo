# 宝藏简介（`description`）编写规范

简介使用 **片段数组**，由 `TreasureDescRichText` 渲染。请在各 `treasures/items/*.js` 中用 `describe(...)` 与 `rarity` / `mult` / `score` / `money` 组合，**不要**再写整段纯字符串。

```js
import { describe, rarity, mult, score, money } from "../treasureDescription.js";

export default {
  // ...
  description: describe(
    "拼写的",
    rarity("传说"),
    "字母在记分时给予",
    mult("+7"),
    "倍率",
  ),
};
```

## 片段类型

| 工厂函数 | `type` | 含义 | 样式 |
|----------|--------|------|------|
| 字符串或隐式 text | `text` | 普通说明文字 | 默认正文色 |
| `rarity('普通' \| '稀有' \| '史诗' \| '传说')` | `rarity` | 稀有度用语 | 常规字重 + 对应稀有度颜色 |
| `mult('+4')` | `mult` | 加法倍率 | 洋红字（无底色，与 `--mult-accent` 同色） |
| `mult('x3')` | `mult` | 乘号倍率（数据里写小写 `x` 即可） | 展示为 **×3**，洋红圆角底 + 白字 |
| `score('+20')` | `score` | 分数加成（含 `+`，可 `+0`） | 分数蓝 |
| `money('3')` | `money` | 金额（**仅写数字**） | 金色 `$` + 数字（与钱包金黄一致） |
| `gain('万能块')` | `gain` | **仅**须加粗的具名棋盘材质短词 | **加粗** + 正文本色；禁止用于整句或抽象效果 |

所有带样式的片段两侧会加少量 **水平间距**，避免与相邻汉字贴死。

## 规则说明

1. **稀有度**  
   仅对「普通 / 稀有 / 史诗 / 传说」四个中文词使用 `rarity(...)`，与棋盘宝石语义一致。

2. **倍率**  
   凡强调倍率的数字均用 `mult(...)`：加法写 `+4`、`+7`、`+0-26` 等；**相乘**写 `x3`、`x1`（界面显示为 **×**，且为洋红底白字标签）。不要把数字留在纯 text 里。

3. **分数**  
   凡「+n 分数」里的 **+n**（含括号内「当前+0」这类与分数同类的 +n）用 `score(...)`。

4. **金钱**  
   - 文案里出现 **`$n` 或「获得 n」且表示金币** 时：用 `money('n')`，只传数字字符串。  
   - 渲染为金色 **`$`** + 数字（若原文是「获得$3」，写成 `describe('…获得', money('3'))`）。

5. **顺序**  
   `describe(a, b, c)` 按阅读顺序拼接；字符串参数会自动变成 `text` 段。

6. **`gain` / `gainBlock`**  
   - `gain(...)` **仅**用于简介里须加粗的具名材质名（如彩虹宝藏主简介中的「万能块」），勿用于强调整句或非材质文案。  
   - `gainBlock` 仅作片段分组，**无**整段加粗；大段说明请用 `describe(...)` 平铺 `text` / chip 段。

## 详情浮层：主简介下的补充分区

商店/已拥有宝藏详情中，主简介下方可有 **具名配饰分区**（标题来自 `treasureAccessories.js`，如「火焰配饰」）及可选的 **`getDetailGainPanel`**。

- **不要**用 `getDetailGainPanel` 解释抽象计分（字后倍率、重播汇总等）。
- **仅**在须单独说明**具名材质/具名配饰产物**时实现；**当前仅 id「77」彩虹**，标题为「万能块」。
- **禁止**用「宝藏配饰」等**类目统称**当分区标题或增益本体。

完整约定见 **`.cursor/rules/treasure-detail-supplement.mdc`**；法术卡下方分区见 **`spell-gain-panel.mdc`**。

## 校验

未知稀有度标签会在开发环境 `console.warn`。新增装备时请对照本表检查一遍再提交。
